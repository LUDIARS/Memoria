# タブ: 要約キュー

## 目的
claude CLI を直列実行する要約キューと埋め込みキューの状態 (実行中・順番待ち・履歴) を可視化する。

## 入力 API

| メソッド | パス | 用途 |
|---------|------|------|
| `GET` | `/api/queue` | 深度・実行中 (両キュー、ヘッダーバッジ用) |
| `GET` | `/api/queue/items` | snapshot — 実行中・queued・history (履歴 50 件) |

## UI 要素

### 「実行中」セクション
- 1 件のみ表示 (head が `running` なら)
- 青の脈動ドット + タイトル + URL + `経過 N.Ns` (リアルタイム更新)
- 何もなければ点線ボックスに「実行中のジョブはありません」

### 「順番待ち N」
- 番号付きリスト (CSS counter で 1, 2, 3... 自動採番)
- 各項目: タイトル + URL

### 「履歴 N」
- 最新 50 件、列: ✓/✗ アイコン (done/error) / タイトル + URL + エラー / 所要時間 + 完了時刻
- 履歴は `setSummary(status='done'|'error')` の度に頭に積まれる

## 動作

- 全タブ共通の 2 秒ポーリングで snapshot を取得 → state 更新
- 自分が active タブのときだけ DOM を再描画 (経過秒の更新)
- メイン用 `summaryQueue` と RAG 用 `embeddingQueue` の両方の状況をマージ表示する

## 制限

- 履歴は in-memory のみ。サーバー再起動で消える (永続化は未実装)
- pending → running の遷移瞬間は「実行中」セクションの内容が点滅する場合あり

## ロードマップ

- 履歴を `dispatch_log` 的な永続テーブルに移して長期保持
- failed ジョブの「retry」ボタン
- 重い (要約 > 60s) ジョブのプロファイル表示
