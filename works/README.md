# works/

作業用スクリプト・実験コード・デバッグ用フィクスチャ置き場。

このフォルダはコミットされる場合もあるが **CI 対象外** で、本番ランタイムからは
import されない。すぐ捨ててよい一時ファイルから、繰り返し使うけど service 本体に
入れたくない補助スクリプトまで自由に使う。

## 想定される置き場所

| サブパス | 用途 |
|---------|------|
| `works/scripts/` | DB の中身を覗く一発スクリプト、再要約バッチ等 |
| `works/fixtures/` | 統合テスト用のサンプル HTML / JSON ペイロード |
| `works/notebooks/` | 仮データ分析・SQL クエリのメモ |
| `works/sandbox/` | API を叩く curl サンプル等 |

## ルール

- **本番コード依存は持ち込まない**: `service/` から `works/` を import するのは禁止
- **個人データはコミットしない**: 実 DB のダンプや HTML ファイルは置かない
- **CI**: `.github/workflows/ci.yml` は `works/` を無視する (syntax check も対象外)
- **退役**: 必要なら定期的に整理して掃除する。歴史を残したい場合は git tag だけ打って中身は削除して可
