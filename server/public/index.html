📌 スマホでブックマークする方法

スマホには Chrome 拡張が無いので、Memoria を PWA としてインストールしてから OS の共有メニューで送ります。

  1. Memoria の URL を開いた状態で、ブラウザのメニュー → 「ホーム画面に追加」 または 「アプリをインストール」
  2. シェアしたいページで 共有 ボタンを押す
  3. 共有先のリストから Memoria を選ぶと自動で保存キューに入ります

iOS Safari は Web Share Target に対応しないので、iOS Shortcut テンプレート を使ってください。

🧩 Chrome 拡張をセットアップ

Memoria はブラウザで開いているページを Chrome 拡張から受け取ります。 この PC で拡張がまだ動いていないようです。下の手順で 1 度だけ設定してください。

  1. Chrome で chrome://extensions を開く
  2. 右上の 「デベロッパーモード」 を ON
  3. 「パッケージ化されていない拡張機能を読み込む」 をクリックし、 このリポジトリの extension/ フォルダを選ぶ
  4. 拡張のオプションを開き、サーバ URL に http://localhost:5180 が入っていることを確認
  5. 適当なページを開いてみてください。数秒後ここの表示が ✓ 拡張 OK に変わります

うまく動かないときは chrome://extensions の Memoria Bookmarker を 🔄 リロード → 確認したいページで F5 で再読込。

                                      📖 設定方法

                                      取り込みたい情報源ごとに必要なステップを実施してください。 どちらも default OFF で、 明示的に enable しないと取込まれません。

                                      📍 A. 位置情報 (OwnTracks GPS) の設定

                                      iPhone の OwnTracks アプリが Tailscale 経由で home PC 同居 Mosquitto に publish → Legatus が subscribe → Memoria の gps_locations に永続化。

                                      1️⃣ Memoria 側で ingest key を発行

                                      「🗺 軌跡」 タブで「ingest key を再生成」 を押す → 32 桁 hex が発行される (= Mosquitto Basic auth password として使える)。

                                      2️⃣ home PC で Mosquitto を起動 (Legatus 同居)

                                      # /etc/mosquitto/mosquitto.conf (抜粋)
listener 1883 0.0.0.0
allow_anonymous false
password_file /etc/mosquitto/passwd
# tailnet 経由のみ受ける場合 (推奨):
# listener 1883 100.x.y.z   (home PC の Tailscale IP)

                                      mosquitto_passwd -c /etc/mosquitto/passwd owntracks で user 作成、 password に上の ingest key を入れる。

                                      3️⃣ Legatus に env vars を渡す

                                      OWNTRACKS_ENABLED=true
OWNTRACKS_MQTT_URL=mqtt://127.0.0.1:1883
OWNTRACKS_MQTT_USERNAME=owntracks
OWNTRACKS_MQTT_PASSWORD=<ingest key>
OWNTRACKS_MQTT_TOPIC=owntracks/+/+
LEGATUS_LOCATION_FLUSH_INTERVAL_MS=300000
LEGATUS_LOCATION_MIN_DISPLACEMENT_M=100

                                      4️⃣ iPhone OwnTracks アプリ

                                      • Mode = MQTT
                                      • Host = home PC の Tailscale IP (例 100.64.0.5)
                                      • Port = 1883
                                      • User = owntracks / Password = ingest key
                                      • Topic format は default (owntracks/<user>/<device>)
                                      • 「Move」 mode 推奨 (バッテリー寄りなら Significant)

                                      5️⃣ オプトアウト

                                      1. iPhone 側 OwnTracks の publish を OFF
                                      2. Legatus の OWNTRACKS_ENABLED=false
                                      3. Memoria 「🗺 軌跡」 タブで ingest key を削除 (= 受信を恒久停止)

                                      🌐 B. DNS / SNI tap (Legatus dnstap) の設定

                                      home PC の dnsmasq が iPhone (Tailscale exit node) からの DNS query log を吐き、 Legatus dnstap が tail → Tailscale で device tag → Memoria に forward。

                                      1️⃣ Legatus dnstap を起動 (env vars)

                                      個人 PC で動かす Legatus に以下の環境変数を渡します。 全項目 default OFF — 明示的に enable する必要があります。

                                      LEGATUS_DNSTAP_ENABLED=true
LEGATUS_DNSMASQ_LOG_PATH=/var/log/dnsmasq.log
LEGATUS_DNSTAP_FLUSH_MS=30000
LEGATUS_DNSTAP_DEDUPE_MS=5000
LEGATUS_TAILSCALE_BIN=tailscale
LEGATUS_TAILSCALE_REFRESH_MS=300000
LEGATUS_DNSTAP_SKIP_DOMAINS=bank.example,hospital.test
LEGATUS_DNSTAP_FORWARD_URL=http://localhost:5180/api/visits/external
                                      各キーの意味
                                      • LEGATUS_DNSTAP_ENABLED — モジュール ON/OFF。 default false
                                      • LEGATUS_DNSMASQ_LOG_PATH — tail 対象の query log。 dnsmasq 側 --log-facility と一致
                                      • LEGATUS_DNSTAP_FLUSH_MS — Memoria への flush 周期 (default 30 秒)
                                      • LEGATUS_DNSTAP_DEDUPE_MS — A/AAAA/HTTPS RR バーストの統合窓 (default 5 秒)
                                      • LEGATUS_TAILSCALE_BIN — tailscale CLI のパス
                                      • LEGATUS_TAILSCALE_REFRESH_MS — IP→hostname map の更新間隔 (default 5 分)
                                      • LEGATUS_DNSTAP_SKIP_DOMAINS — カンマ区切り skip 一覧 (suffix match)
                                      • LEGATUS_DNSTAP_FORWARD_URL — この Memoria の /api/visits/external

                                      2️⃣ dnsmasq の設定 (/etc/dnsmasq.conf)

                                      log-queries
log-facility=/var/log/dnsmasq.log
interface=tailscale0
interface=lo
bind-interfaces
domain-needed
bogus-priv

                                      tailscale0 インターフェイスでのみ listen、 公開しない構成。 logrotate も忘れず。

                                      3️⃣ Tailscale 設定

                                      home PC (Legatus 同居)
                                      • tailscale up --advertise-exit-node
                                      • admin console で 承認
                                      • 必要なら --advertise-routes=192.168.x.0/24
                                      Tailscale admin console
                                      • DNS → Override local DNS ON
                                      • Nameservers に home PC の Tailscale IP を登録
                                      • MagicDNS ON
                                      iPhone Tailscale アプリ
                                      • Use Tailscale DNS ON
                                      • Exit Node = home PC を選択

                                      4️⃣ iPhone iOS 側

                                      ⚠ 設定 → Apple ID → iCloud → Private Relay を OFF (該当 Wi-Fi のみで可)。 ON のままだと DNS が Apple/Cloudflare に飛んで home resolver を bypass し、 ログが空になります。
                                      • Tailscale 接続を常時 ON
                                      • Wi-Fi → Configure DNS = Automatic
                                      • Tailscale exit node を home PC にしている時のみ home dnsmasq 経由

                                      5️⃣ オプトアウト

                                      1. デバイス側: iPhone Tailscale を切る or exit node を外す → 即座に取込停止
                                      2. module 設定: LEGATUS_DNSTAP_SKIP_DOMAINS に銀行 / 医療 / 内部系を列挙
                                      3. 全体停止: LEGATUS_DNSTAP_ENABLED=false (default 値)

                                          CLI バイナリパス (PATH に通っていない場合のみ指定)

                                          🐚 Bash (Windows のみ)

                                          Claude CLI は内部で bash を呼び出します。Windows でデスクトップアプリから動かすときだけ git-bash の絶対パスを指定してください (例: C:\Program Files\Git\bin\bash.exe)。空欄なら環境変数 CLAUDE_CODE_GIT_BASH_PATH を尊重します。

                                          OpenAI API

                                          📤 エクスポート / 📥 インポート

                                          ブックマークを JSON で書き出し / 取り込み (URL 重複はスキップ)。エクスポートはチェックを入れたカードのみ。

                                          📝 日記の常設メモ

                                          日記生成時に毎回プロンプトに添える「常設メモ」。プロジェクトの背景・自分の役割・生成ルール (例: Memoria は LUDIARS の個人ツール。曖昧な作業は推測で断定して書く) などを書いておくと、毎日の日記の作業内容 / ハイライトに反映されます。空欄なら何も追加しません。

                                          🖥 デスクトップアプリ

                                          この Memoria は Electron デスクトップアプリの中で動いています。 ウィンドウの × はトレイ最小化、 サーバは常駐し続けます (Chrome 拡張 / モバイル PWA からのアクセスは継続)。 完全終了はトレイ右クリック → 「Memoria を終了」。

                                          ON にすると --hidden フラグでウィンドウなし起動 → トレイアイコンだけ表示されます。 普段は OS スタートアップに登録するこの方式で十分。 「ログアウト中も動かしたい」 ような用途は将来 Windows サービス化を検討します (別 PR)。

                                          🧹 メンテナンス

                                          過去のアクセス記録に出てきたドメインのうち、まだドメイン辞書に登録されていないものを fetch + 分類キューに積みます。 force にすると既存行も再分類します (user_edited 列は保護)。

                                          📱 通知 (WebPush)

                                          日記生成完了などをこの端末に通知します。 iOS は 「ホーム画面に追加」 して PWA として開いた状態でないと 通知許可ダイアログが出ません (16.4+ の仕様)。

                                            🧍 プロファイル (適正カロリー計算用)

                                            日記のカロリーバランス分析で使用。 未設定だと分析されません。

                                            🛠 ランタイム情報 (読み取り専用)

                                            これらは起動時に決まる値で、変更には再起動が必要です。デスクトップアプリから自動で渡されます。

                                            🌐 マルチサーバ (Memoria Hub)

                                            辞書 / ディグ / ブックマークを共有できるハブを複数登録できます。各エントリは独立した Cernere ログインを持ち、左の topbar スイッチで個別 ON/OFF (複数同時 ON 可)。
